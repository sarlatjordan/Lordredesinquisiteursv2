Gestion des membres

Accessible via /admin/gestion-membres. La page contient deux onglets : Promotions (Sage) et Évaluations de rang (MI+).

Promotions Sage

Pour promouvoir un membre, ouvre sa fiche dans l'onglet Promotions, sélectionne le nouveau rang, et valide. La promotion déclenche automatiquement :

→
Mise à jour du rang en base
Le profil passe au nouveau rang, les droits RLS sont immédiatement actifs.
→
Notification au membre
Le membre reçoit une notification in-app et une push notification si activées.
→
Entrée dans l'historique
La promotion apparaît dans le feed d'activité global et dans l'historique de la fiche membre.

Évaluations de rang MI+

Les évaluations formalisent la validation des étapes de rang. Chaque évaluation est associée à un type d'épreuve (combat, logistique, leadership, etc.) et à un résultat (réussi / en attente / échoué).

Pour créer une évaluation : ouvre la fiche d'un membre → section Évaluations → bouton + Nouvelle évaluation. Remplis le type, la date, et tes observations.

Absences MI+

Les absences déclarées par les membres apparaissent dans l'onglet Absences de /admin/gestion-membres. Tu peux voir qui est absent, pour quelle durée, et le motif.

ℹ️
Les Sages reçoivent une notification à chaque nouvelle déclaration d'absence. Aucune action admin n'est requise — c'est informatif.
📋

Candidatures — pipeline Kanban

Le pipeline de candidatures (/admin/gestion-membres → onglet Candidatures) suit les postulants de la réception à la décision finale via 4 colonnes.

📥 Reçues
Formulaire /recrutement soumis. Pas encore examiné.
🔍 En discussion
Candidature vue, en cours d'évaluation par le Conseil.
✅ Acceptées
Invitation envoyée. Devient Aspirant.
❌ Refusées
Candidature rejetée. Archivée.

Traiter une candidature

1
Ouvrir la fiche candidature
Clique sur une carte pour voir : pseudo RSI, message de motivation, date de soumission, profil SC (si renseigné).
2
Déplacer dans le pipeline
Utilise les boutons de statut pour faire avancer la candidature. Ajoute un commentaire interne si besoin.
3
Accepter → Invitation
Passer en "Accepté" ne crée pas automatiquement le compte — envoie manuellement l'invitation Supabase Auth depuis le dashboard Supabase ou via email.
💡
Le badge compteur sur la carte "Candidatures" dans /admin indique le nombre de candidatures en statut reçu non traitées.
⚡

Points & récompenses

Le système de points récompense la participation des membres. Les points s'accumulent et contribuent à l'éligibilité pour les promotions.

Attribuer des points Inquisiteur+

Sur la fiche d'un membre (/membres/[username]), section Points → bouton Attribuer des points. Sélectionne la raison (participation op, événement, contribution wiki, etc.) et la valeur.

RaisonPoints typiques
Participation opération+10 à +25
Organisation événement+15 à +30
Contribution wiki+5 à +15
Recrutement validé+20
Étape onboarding+10 (automatique)
Bonus complétion rang+25 à +80 (automatique)

Audit des attributions Sage

Accessible via /admin/points — liste les 200 dernières attributions avec l'auteur, le membre récompensé, la raison et la date. Permet de détecter les attributions anormales.

🐛

Rapports de bug — /admin/bugs

Les Maîtres Inquisiteurs+ voient tous les rapports de bug soumis par les membres. Les Sages reçoivent une notification à chaque nouveau rapport.

Gérer un rapport

Clique sur une ligne pour l'ouvrir. Tu peux :

🔄
Changer le statut
Ouvert → En cours → Résolu → Fermé
📝
Ajouter une note
La note est visible par le membre qui a soumis le rapport. Explique le fix, le workaround, ou pourquoi c'est fermé.
🔥
Les rapports Critique sont identifiés par une icône flamme. Traite-les en priorité — ils signalent généralement un problème de sécurité ou une perte de données.
💬

Synchronisation Discord

INQFR synchronise les événements dans les deux sens avec Discord.

Site → Discord (création d'événement)

Lors de la création d'un événement sur INQFR, coche "Publier sur Discord" pour créer automatiquement un Scheduled Event Discord. L'événement apparaît dans l'onglet Événements du serveur.

ℹ️
Le bot doit avoir la permission MANAGE_EVENTS sur le serveur. Vérifie les rôles du bot si la publication échoue.

Discord → Site (Make.com polling)

Make.com scrute les événements Discord toutes les 15 minutes et envoie un POST à /api/discord/events. Les événements Discord sont upsertés sur INQFR par leur discord_event_id. Les statuts sont mappés automatiquement :

Statut DiscordStatut INQFR
1 — SCHEDULEDplanifie
2 — ACTIVEen_cours
3 — COMPLETEDtermine
4 — CANCELEDannule

Notifications Discord sur les événements

Lors de la création d'un événement, INQFR poste automatiquement un message dans le canal configuré avec des mentions de rang (@Gardien, @Inquisiteur…) selon le rang minimum de l'événement.

Journal de guerre Sage

Le journal de guerre (/admin/journal) est le registre officiel des opérations et batailles marquantes de l'Ordre. Il alimente aussi la section "Journal" de la landing page publique.

Créer une entrée

Chaque entrée contient : titre, date de l'opération, résultat (victoire / défaite / neutre), description markdown complète, et une image optionnelle. Le contenu supporte les éditeurs markdown avec toolbar (H1/H2, gras, italique, listes).

ℹ️
Les Inquisiteurs peuvent accéder à /admin/journal directement depuis la sidebar (section "Administration") même sans accès au panneau admin complet.
🖼

Galerie Sage

La galerie (/admin/galerie) permet d'uploader des captures d'écran et médias Star Citizen dans le Storage Supabase. Ils apparaissent sur la galerie publique /galerie.

Upload

Glisse-dépose ou sélectionne les fichiers (JPG, PNG, WebP). Chaque upload peut avoir un titre et une description. Les fichiers sont stockés dans le bucket Supabase gallery.

⚠️
La galerie est publique — ne pas uploader de contenu interne sensible (plans d'ops, positions, composition de flotte stratégique).
🔐

Activité globale Sage

La page /admin/activite affiche un journal complet de toutes les actions enregistrées dans l'Ordre : connexions, promotions, attributions de points, créations d'op, transactions logistiques…

Utile pour détecter des comportements anormaux (attributions de points excessives, connexions inhabituelles) et auditer l'activité en cas de litige.

🔑

Rotation des clés Sage

En cas de fuite suspectée ou de départ d'un membre privilégié, procéder à la rotation des clés Supabase immédiatement.

1
Révoquer dans Supabase Dashboard
Settings → API → Regenerate. L'ancienne clé est invalidée instantanément.
2
Mettre à jour Vercel
Settings → Environment Variables → mettre à jour NEXT_PUBLIC_SUPABASE_ANON_KEY et SUPABASE_SERVICE_ROLE_KEY.
3
Redéployer
Vercel Dashboard → Redeploy (without cache). Le déploiement prend 1 à 3 minutes.
4
Vérifier
Login, magic link, aucun 401/403 dans les logs Vercel. Auditer Supabase Logs → API sur les 24h précédentes.
🚨
Urgence service_role compromise : régénérer immédiatement → redéployer → auditer les logs Supabase sur 24h. La clé service_role bypass toutes les RLS — une fuite est critique.
